## describe / it / test

roadtest uses a familiar Jest-compatible DSL for structuring tests. Import everything from `roadtest`.

```ts
import { describe, it, test, expect } from "roadtest";

describe("MyComponent", () => {
  it("does something", async () => {
    expect(1 + 1).toBe(2);
  });

  test("alias for it", async () => {
    expect(true).toBeTruthy();
  });
});
```

Test callbacks are always **async**. `await` any `render()` or `fireEvent` call.

---

## describe

```ts
function describe(name: string, fn: () => void): void;
```

Groups related tests. The `name` appears as the suite label in the test tree. `describe` blocks can be nested to any depth.

```ts
describe('Form', () => {
  describe('validation', () => {
    it('rejects empty fields', async () => { ... })
    it('rejects invalid email', async () => { ... })
  })

  describe('submission', () => {
    it('sends the correct payload', async () => { ... })
  })
})
```

### describe.only

```ts
describe.only(name: string, fn: () => void): void
```

Marks this suite as the only one to run in the file. All other suites are skipped. Useful when debugging a single feature.

```ts
describe.only('focused suite', () => {
  it('runs', async () => { ... })
})

describe('skipped suite', () => {
  it('does not run', async () => { ... })
})
```

### describe.each

```ts
describe.each<T extends unknown[]>(cases: T[]): (
  nameTemplate: string,
  fn: (...args: T) => void
) => void
```

Runs the same describe block for each row of inputs. Useful for running an entire suite of tests across multiple configurations (themes, locales, user roles, etc.).

Use `%s`, `%d`, `%i`, or `%o` as placeholders in the name template.

```ts
describe.each([
  ['light'],
  ['dark'],
])('Button — %s theme', async (theme) => {
  it('renders without crashing', async () => {
    const { getByRole } = await render(<Button theme={theme} label="Save" />)
    expect(getByRole('button')).toBeTruthy()
  })

  it('shows the label', async () => {
    const { getByText } = await render(<Button theme={theme} label="Save" />)
    expect(getByText('Save')).toBeTruthy()
  })
})
```

This produces two separate suites — `Button — light theme` and `Button — dark theme` — each with the full set of tests.

## it / test

```ts
function it(name: string, fn: () => void | Promise<void>, options?: TestOptions): void;
function test(name: string, fn: () => void | Promise<void>, options?: TestOptions): void;
```

`test` is an alias for `it` — they are identical. Use whichever reads better for your test.

### TestOptions

```ts
interface TestOptions {
  timeout?: number; // ms — overrides the global setTestTimeout() for this test only
}
```

Pass a third argument to override the default timeout for a specific test:

```ts
it(
  "waits for slow animation",
  async () => {
    // ...
  },
  { timeout: 10000 },
);
```

### Duplicate name detection

Registering two tests with the same name in the same suite will log a console warning and only the first definition will run. Rename the second test to fix it.

### it.only

```ts
it.only(name: string, fn: () => void | Promise<void>, options?: TestOptions): void
```

Runs only this test within its suite. All other `it` / `test` calls in the same file are skipped.

```ts
describe("Button", () => {
  it.only("renders label", async () => {
    // only this test runs
  });

  it("handles click", async () => {
    // skipped
  });
});
```

### it.skip

```ts
it.skip(name: string, fn: () => void | Promise<void>): void
```

Marks a test as skipped. It appears in the test tree with a skipped status and does not run.

```ts
it.skip("not implemented yet", async () => {
  // body is never executed
});
```

### it.each

```ts
it.each<T extends unknown[]>(cases: T[]): (
  nameTemplate: string,
  fn: (...args: T) => void | Promise<void>
) => void
```

Runs the same test body for each row of inputs. `test.each` is also available as an alias.

Use `%s`, `%d`, `%i`, or `%o` as placeholders in the name template — they are replaced with the corresponding argument values in order.

```ts
it.each([
  [1, 1, 2],
  [2, 3, 5],
  [10, -2, 8],
])("adds %d + %d = %d", async (a, b, expected) => {
  expect(a + b).toBe(expected);
});
```

This produces three separate tests:

- `adds 1 + 1 = 2`
- `adds 2 + 3 = 5`
- `adds 10 + -2 = 8`

Each test appears independently in the test tree and can be inspected separately in the UI.

```ts
// Works with objects too — use %o for object substitution
it.each([
  [{ name: 'Alice', role: 'admin' }],
  [{ name: 'Bob',   role: 'viewer' }],
])('renders user %o', async (user) => {
  const { getByText } = await render(<UserCard user={user} />)
  expect(getByText(user.name)).toBeTruthy()
})
```

## Lifecycle hooks

Suite-scoped setup and teardown functions. Import from `roadtest` and call them **inside** a `describe` block.

```ts
import { describe, it, beforeAll, afterAll, beforeEach, afterEach } from "roadtest";
```

### beforeAll

```ts
beforeAll(fn: () => void | Promise<void>): void
```

Runs once before any test in the enclosing `describe` block. Use for expensive one-time setup (seeding a database, building a fixture, starting a server).

### afterAll

```ts
afterAll(fn: () => void | Promise<void>): void
```

Runs once after all tests in the enclosing `describe` block have finished. Use for one-time teardown.

### beforeEach

```ts
beforeEach(fn: () => void | Promise<void>): void
```

Runs before each individual test in the enclosing `describe` block. Runs after any global `registerBeforeTestHook` hooks.

### afterEach

```ts
afterEach(fn: () => void | Promise<void>): void
```

Runs after each individual test in the enclosing `describe` block. Runs after any global `registerAfterTestHook` hooks.

### Example

```ts
describe("UserService", () => {
  let userId: string;

  beforeAll(async () => {
    userId = await db.createUser({ name: "Alice" });
  });

  afterAll(async () => {
    await db.deleteUser(userId);
  });

  beforeEach(() => {
    resetRequestMocks();
  });

  afterEach(() => {
    clearLocalStorage();
  });

  it("fetches the user", async () => {
    const user = await fetchUser(userId);
    expect(user.name).toBe("Alice");
  });

  it("updates the user", async () => {
    await updateUser(userId, { name: "Bob" });
    const user = await fetchUser(userId);
    expect(user.name).toBe("Bob");
  });
});
```

These hooks are scoped to their `describe` block. For file-level or global setup (e.g. starting MSW), use `registerBeforeTestHook` / `registerAfterTestHook` in `.roadtest/setup.ts` instead — see [registerAfterTestHook](/docs/after-test-hook).
