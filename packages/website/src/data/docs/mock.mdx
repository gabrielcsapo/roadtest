## mock / unmock

Replace module imports with controlled fakes. `mock()` calls are automatically hoisted before imports by the roadtest Vite plugin — the same pattern as Vitest's `vi.mock()`.

### Type signatures

```ts
function mock(moduleId: string, factory?: () => Record<string, unknown>): void;
function unmock(moduleId: string): void;
function clearAllMocks(): void;
```

---

## mock

```ts
mock(moduleId: string, factory?: () => Record<string, unknown>): void
```

Registers a module mock. The call is hoisted to the top of the file before any imports are resolved, so the factory result is what your imports receive.

`mock()` returns `void`. Your `import` statements are what give you access to the mocked exports — they are redirected to the factory after hoisting.

```ts
import { describe, it, expect, render, mock } from 'roadtest'
import { useFeatureFlag } from '../hooks/useFeatureFlag'

mock('../hooks/useFeatureFlag', () => ({
  useFeatureFlag: () => true,
}))

describe('FeatureGate', () => {
  it('shows content when flag is on', async () => {
    const { getByText } = await render(<FeatureGate flag="new-ui" />)
    expect(getByText('New UI')).toBeTruthy()
  })
})
```

If you omit the factory, the module is auto-mocked with an empty object (`{}`).

## Spy on calls

Every function exported from a mock factory is automatically wrapped with a spy. The spy records every call — use the built-in spy matchers or read `_spyCalls` directly.

```ts
import { fetchUser } from '../api/user'

mock('../api/user', () => ({
  fetchUser: async (id: string) => ({ id, name: 'Test User' }),
}))

it('calls fetchUser with the right id', async () => {
  await render(<UserProfile id="abc" />)

  expect(fetchUser).toHaveBeenCalled();
  expect(fetchUser).toHaveBeenCalledTimes(1);
  expect(fetchUser).toHaveBeenCalledWith("abc");
  expect(fetchUser).toHaveBeenLastCalledWith("abc");
})
```

You can also read the raw call log directly:

```ts
interface SpyCall {
  args: unknown[];
  result: unknown;
  threw: boolean;
}

expect(fetchUser._spyCalls[0].args[0]).toBe("abc");
```

## Configurable mock functions

Auto-wrapped spies record calls but don't let you change what they return. When you need to control the return value or implementation per-test, use `vi.fn()` (vitest-compat) or `createMockFn()` (native) in the factory:

```ts
import { vi } from 'vitest'
import { fetchUser } from '../api/user'

mock('../api/user', () => ({
  fetchUser: vi.fn(),
}))

it('returns mocked user', async () => {
  fetchUser.mockResolvedValue({ id: '1', name: 'Alice' })
  await render(<UserProfile id="1" />)
  expect(fetchUser).toHaveBeenCalledWith('1')
})

it('handles errors', async () => {
  fetchUser.mockRejectedValue(new Error('not found'))
  await render(<UserProfile id="bad" />)
  // ...
})
```

### Mock function methods

| Method                       | Description                             |
| ---------------------------- | --------------------------------------- |
| `mockReturnValue(val)`       | Always return `val`                     |
| `mockReturnValueOnce(val)`   | Return `val` for the next call only     |
| `mockImplementation(fn)`     | Replace the implementation              |
| `mockImplementationOnce(fn)` | Use `fn` for the next call only         |
| `mockResolvedValue(val)`     | Return `Promise.resolve(val)`           |
| `mockRejectedValue(val)`     | Return `Promise.reject(val)`            |
| `mockResolvedValueOnce(val)` | Resolved promise for the next call only |
| `mockRejectedValueOnce(val)` | Rejected promise for the next call only |
| `mockReturnThis()`           | Return `this`                           |
| `mockClear()`                | Reset call records, keep implementation |
| `mockReset()`                | Reset call records and implementation   |
| `mockRestore()`              | Restore original (for `spyOn` only)     |
| `mockName(name)`             | Set a display name                      |

### mock.calls / mock.results

Mock functions also expose a `.mock` property compatible with the Vitest / Jest API:

```ts
const spy = vi.fn();
spy("a");
spy("b");

spy.mock.calls; // [['a'], ['b']]
spy.mock.results; // [{ type: 'return', value: undefined }, ...]
spy.mock.instances; // [undefined, undefined]  (set when called with `new`)
```

## unmock

```ts
unmock(moduleId: string): void
```

Removes a specific module mock. The next import of that module will use the real implementation.

```ts
import { unmock } from "roadtest";

unmock("../api/user");
```

---

## clearAllMocks

```ts
clearAllMocks(): void
```

Removes all registered mocks and resets the call log. Useful in an `afterEach` / `afterAll` hook to ensure clean state between test files.

```ts
import { clearAllMocks, registerAfterTestHook } from "roadtest";

registerAfterTestHook(() => {
  clearAllMocks();
});
```

---

## How hoisting works

The roadtest Vite plugin statically rewrites your test file so that `mock()` calls execute before any module imports are resolved. This means:

- You **cannot** reference variables defined outside the factory inside the factory — the factory runs before the rest of the module is evaluated.
- The `import` statements at the top of the file will receive the mocked versions automatically.
- Mocks are scoped to the test file that registers them.
