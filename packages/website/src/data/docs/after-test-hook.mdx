## registerAfterTestHook

Run custom logic after **every** test in the project completes. Designed for use in `.fieldtest/setup.ts` — these hooks are global and apply across all test files.

```ts
import { registerAfterTestHook, currentTest } from "fieldtest";

registerAfterTestHook(async () => {
  if (!currentTest) return;

  // e.g. attach performance metrics
  console.log(`[${currentTest.name}] render count: ${getRenderCount()}`);
});
```

`currentTest` is a variable (not a function) — read it directly. It holds the active `TestCase` during a test run, or `null` outside of one.

The hook runs in the same environment as the test (browser or Node). In browser mode, you can access the DOM. In Node mode, `currentTest` still gives you full test metadata.

### All global hooks

```ts
import {
  registerBeforeTestHook,
  registerAfterTestHook,
  registerBeforeDisplayHook,
  registerAfterDisplayHook,
} from 'fieldtest'

// Runs before each test (all files)
registerBeforeTestHook(async () => { ... })

// Runs after each test (all files)
registerAfterTestHook(async () => { ... })

// Runs before rendering in the browser UI (display mode only)
registerBeforeDisplayHook(async () => { ... })

// Runs after rendering in the browser UI (display mode only)
registerAfterDisplayHook(async () => { ... })
```

### Suite-scoped hooks

For setup and teardown that should only apply within a specific `describe` block, use the DSL hooks instead — `beforeAll`, `afterAll`, `beforeEach`, and `afterEach`. These are imported from `fieldtest` and called directly inside a `describe`. See [describe / it / test](/docs/describe-it) for full documentation and examples.
