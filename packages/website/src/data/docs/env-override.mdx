## Environment Override

By default, test files run in whatever environment roadtest was invoked in (browser or Node). You can force a specific file to always run in Node with a comment at the top:

```ts
// @roadtest-env node
import { describe, it, expect } from "roadtest";

describe("pure logic", () => {
  it("sorts correctly", () => {
    expect([3, 1, 2].sort()).toEqual([1, 2, 3]);
  });
});
```

This is useful for tests that involve Node-only APIs or where a DOM is unnecessary overhead.
